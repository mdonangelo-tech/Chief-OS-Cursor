# ChiefOS Decisions

Use this folder for lightweight records of durable product, architecture, or operational choices. Do not record every small implementation detail.

## When To Record

- The decision changes architecture, cost, data flow, deployment, or a stable user workflow.
- There were multiple plausible options and the trade-off matters later.
- Future agents are likely to revisit the same question.

## Filename

Use:

```text
YYYY-MM-DD-short-decision.md
```

## Template

```md
# Decision: Short Title

## Status

Proposed | Accepted | Superseded

## Context

## Decision

## Alternatives Considered

## Consequences

## Rollback Or Migration Path
```
